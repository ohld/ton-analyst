# Address Investigation Router

Use this router for unlabelled TON addresses. Load the smallest next page for the current task.

## Routes

| Need | Read |
|------|------|
| Decide whether an address should be labelled | [label-submission.md](label-submission.md) |
| Investigate funders, flows, comments, labels, and TONAPI fields | [wallet-investigation.md](wallet-investigation.md) |
| Use the local context-efficient TONAPI wrapper | [../cli.md](../cli.md) |
| Known entities, categories, CEX, sybil, and Fragment addresses | [labels.md](labels.md) |

## Rule

Do not label personal wallets or addresses with only generic funding provenance. Label an entity-operated address only when direct proof exists or at least two independent indirect proofs agree.
